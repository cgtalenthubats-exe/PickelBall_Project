## 2. ⚠️ ข้อสังเกตสำคัญ — สเปคนี้ยัง "ไม่ครบ" สำหรับเว็บสมัครงานจริง

สเปคของลูกค้าเป็นแค่ **เว็บ "ประกาศ/ค้นหางาน" + CMS** เท่านั้น
**ไม่มีขั้นตอน "สมัครงานจริง" (Apply)** ซึ่งเป็นหัวใจของเว็บ careers สมัยใหม่ ควรเสนอเพิ่ม:

- ปุ่ม/ฟอร์มสมัครงาน + อัปโหลดเรซูเม่ (PDF)
- โปรไฟล์ผู้สมัคร / บัญชีผู้ใช้ / ประวัติการสมัคร
- อีเมลแจ้งเตือน "ได้รับใบสมัครแล้ว"
- หลังบ้านดูรายชื่อผู้สมัครต่อ 1 ตำแหน่ง (mini-ATS)

## 3. ฟังก์ชันมาตรฐานที่เว็บสมัยใหม่ "ควรมี" (เพิ่มเติมจากสเปค)

### ประสบการณ์ผู้ใช้ / การค้นหา
- ค้นหาแบบ real-time + **pagination + sort** (ใหม่สุด/เกี่ยวข้องสุด)
- บันทึกงานที่สนใจ (Save/Bookmark) + **Job Alert** (สมัครรับอีเมลเมื่อมีงานใหม่ตรงเงื่อนไข)
- แชร์งานลงโซเชียล / คัดลอกลิงก์
- Responsive มือถือ 100% + รองรับ dark mode

### SEO & การค้นเจอ
- **Structured Data `JobPosting` (schema.org)** → ขึ้น Google for Jobs ฟรี (สำคัญมากสำหรับเว็บหางาน)
- Sitemap / robots.txt / meta OG / URL อ่านง่าย
- Server-side rendering เพื่อ SEO

### กฎหมาย & ความปลอดภัย (จำเป็นในไทย)
- **PDPA**: หน้ายินยอม (consent) เก็บข้อมูลส่วนบุคคล + นโยบายความเป็นส่วนตัว + Cookie consent
- reCAPTCHA / rate limit กันบอทสแปมใบสมัคร
- HTTPS, เข้ารหัสข้อมูลผู้สมัคร, กำหนดอายุการเก็บข้อมูล
- WCAG (Accessibility) ขั้นพื้นฐาน

### หลังบ้าน (Admin)
- **Role-based access** (Super admin / HR / Editor) + Audit log
- Dashboard สรุปตัวเลข (จำนวนงานเปิด, ผู้สมัคร, ยอดเข้าชม)
- Rich text editor สำหรับ JD, จัดการหมวด/แผนก/สถานที่แบบ master data

### วิศวกรรม / ปฏิบัติการ
- Google Analytics 4 / conversion tracking
- CDN + caching รูปภาพ, image optimization
- CI/CD, backup ฐานข้อมูลอัตโนมัติ, monitoring/uptime alert
- Environment แยก non-prod / prod (ตรงกับที่สเปคระบุ UAT/PVT/Go-live)

## 4. เราต้องใช้ Tech Stack ตามสเปคนี้ไหม? — **ไม่จำเป็น**

**Tech stack (Vue + Laravel + Java Spring Boot + MySQL) เป็น "ข้อเสนอของคู่แข่ง" ไม่ใช่ข้อบังคับของลูกค้า** เว้นแต่ Central Group จะระบุเป็นเงื่อนไขจัดซื้อ (เช่นต้องเข้ากับทีม IT เดิม/มาตรฐานองค์กร) โดยปกติเจ้าของงานสนใจ *ผลลัพธ์และ SLA* มากกว่าภาษาที่ใช้

**ข้อสังเกต:** สแตกที่คู่แข่งเสนอมี **3 ภาษา/เฟรมเวิร์กในระบบเดียว** (Vue, Laravel, Java) ซึ่ง**ซับซ้อนและแพงในการดูแลเกินจำเป็น** — โดยทั่วไปแยก Laravel *หรือ* Spring Boot อย่างใดอย่างหนึ่งก็พอ

**ข้อได้เปรียบของเรา:** repo ปัจจุบันเป็น **Next.js + i18n (2 ภาษา) + ระบบ admin + auth อยู่แล้ว** เราจึงเสนอสแตกเดียวจบ (full-stack เดียว) ซึ่ง *ถูกกว่า เร็วกว่า ดูแลง่ายกว่า*:

| | สแตกคู่แข่ง | สแตกที่เราแนะนำ |
|---|---|---|
| Front-end | Vue.js + Laravel | **Next.js (React) + Tailwind** |
| Back-end/API | Java Spring Boot | **Next.js API routes / Node** (หรือ Laravel ถ้าลูกค้าถนัด PHP) |
| DB | MySQL | **PostgreSQL/MySQL** |
| i18n | ต้องทำเพิ่ม | **มีโครง `[locale]` แล้ว** |
| Deploy | Docker VM | Docker / Vercel |

> **ข้อเสนอแนะ:** ยื่น 2 ทางเลือก — (1) สแตกของเราซึ่งคุ้มกว่า, (2) ถ้าลูกค้ายืนยันต้องใช้สแตกตามสเปค เราทำได้แต่ราคาสูงขึ้น (ดูข้อ 6)

## 6. ราคา (เปรียบเทียบกับคู่แข่ง 120,000 บาท)

ใบเสนอราคาคู่แข่ง:
| รายการ | ราคา (บาท) |
|--------|-----------|
| ค่าพัฒนาระบบ (หน้าบ้าน + หลังบ้าน + API) | 90,000 |
| ค่าดูแลระบบ 1 ปี (MA) | 20,000 |
| Host/VM 1 ปี (SSD 180GB/RAM 8/CPU 4) | 10,000 |
| **รวม** | **120,000** |

> **ความเห็น:** 90,000 บาท สำหรับเว็บ careers 2 ภาษา + CMS เต็ม + API ถือว่า **ต่ำมาก** (≈ ราคาฟรีแลนซ์/ทีมเล็ก) แทบไม่รวม buffer สำหรับ Apply flow, SEO, PDPA เราจึงตั้งราคาได้ตามมูลค่างานจริง

### ทางเลือก A — ตรงสเปค (เท่าที่ลูกค้าขอ) เพื่อ "สู้ราคา"
| รายการ | ราคา (บาท) |
|--------|-----------|
| ค่าพัฒนา (3 ส่วนตามสเปค, สแตกเรา) | 110,000 – 130,000 |
| MA 1 ปี | 20,000 – 25,000 |
| Host/VM 1 ปี | 10,000 – 15,000 |
| **รวม** | **~140,000 – 170,000** |

### ทางเลือก B — สเปคครบเว็บสมัยใหม่ (รวม Apply/SEO/PDPA/Dashboard) *แนะนำ*
| รายการ | ราคา (บาท) |
|--------|-----------|
| ค่าพัฒนา (สเปค + ฟังก์ชันมาตรฐานข้อ 3) | 180,000 – 240,000 |
| MA 1 ปี | 30,000 – 40,000 |
| Host/VM 1 ปี | 12,000 – 18,000 |
| **รวม** | **~220,000 – 300,000** |

### ทางเลือก C — ตรงสแตกคู่แข่งเป๊ะ (Vue + Laravel + Java Spring Boot)
- บวกเพิ่มจากทางเลือก A/B ประมาณ **+20–30%** เพราะดูแล 3 เฟรมเวิร์ก + ต้องมีคนถนัดทั้ง PHP และ Java

**ค่าใช้จ่ายที่ลูกค้ารับผิดชอบเอง (ไม่รวมในราคา):**
- Google Maps API (ตามการใช้จริง — Central Group เปิดบัญชีเอง ตามหมายเหตุ 5.1)
- Domain (careers.centralgroup.com — ลูกค้าต้อง unmap host เดิมก่อน)

**เงื่อนไขชำระเงินที่แนะนำ:** งวด 1 = 40–50% ตอนเริ่ม · งวด 2 = 40% ตอน UAT ผ่าน · งวด 3 = 10–20% ตอน Go-live (แทนที่จะ 50/50 แบบคู่แข่ง เพื่อลดความเสี่ยงกระแสเงินสด)

## 7. สิ่งที่ควรถามลูกค้าให้ชัดก่อนยืนราคา
1. **บังคับใช้ tech stack ตามสเปคไหม** หรือเปิดให้เราเลือกได้?
2. ต้องมี **ขั้นตอนสมัครงานจริง (Apply + resume)** ไหม หรือแค่ประกาศงาน?
3. "ระบบสร้างงาน" ที่จะยิงเข้ามาทาง API (2.10) — สเปค/รูปแบบข้อมูลเป็นอย่างไร (REST/JSON? auth?)
4. จำนวนตำแหน่งงาน/ผู้เข้าชมโดยประมาณ (มีผลต่อ sizing ของ VM)
5. ต้องรองรับ PDPA / เก็บข้อมูลผู้สมัครตามข้อกำหนด compliance ระดับองค์กรไหม
6. ใช้ domain/subdomain เดิมหรือใหม่ และใครดูแล DNS
